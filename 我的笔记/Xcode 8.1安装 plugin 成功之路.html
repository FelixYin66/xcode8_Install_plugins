安装须知：

1.插件需要添加UUID的原因？
由于从Xcode 5开始，苹果要求加入UUID证书从而保证插件的稳定性。因此Xcode版本更新之后需要在每个安装的插件中的plist文件中添加 Xocde 的 UUID，不添加的话，插件安装成功，也不工作（弹出 load bundle属正常）

2.如何查看 Xcode的 DVTPlugInCompatibilityUUID ？
在终端执行 defaults read /Applications/Xcode.app/Contents/Info DVTPlugInCompatibilityUUID

3.如何查看安装插件路径？~/Library/Application Support/Developer/Shared/Xcode/Plug-ins 

4.下面的操作都是基于 Xcode 8.1 安装在  /Applications/Xcode.app 路径下（应用程序/Xcode.app），如果不是安装在 /Applications/Xcode.app 路径下的话有两种解决方案： 1.将Xcode.app 剪切到 Applications 目录下，这种最简单，最安全，不至于出现问题   2. 如果不剪切的话，需要 将路径切换成 你Xcode.app 的所在路径  ❤️❤️❤️❤️❤️❤️

Step1 :
           关闭Xcode，打开钥匙串创建一个 证书 如下图：





按照上图操作后创建了 “XcodeSign” 签名 


Step2:
 在终端运行如下命令：sudo codesign -f -s XcodeSign /Applications/Xcode.app

到这一步，Xcode8 已经可以安装Alcatraz <如果不会安装 Alcatraz 的话，可以去找一个教程，我就不多说了>

Step3:
   安装完Alcatraz 插件后，需要进行一个很重要的操作，在 Alcatraz 中的plist添加Xcode 的 UUID

如何找到 Alcatraz 的 plist 文件，可使用快捷键 (COMMAND + SHIFT + G + ~/Library/Application Support/Developer/Shared/Xcode/Plug-ins）就可以看到Alcatraz 了，看下图：




接着“右击”Alcatraz 显示包内容，看到 contents 文件夹，进入此文件夹可看到 info.plist 配置文件，使用Xcode 或者其他工具打开就可以了，编辑 “DVTPlugInCompatibilityUUIDs” 选项后添加 Xcode 的 UUID，看下图：




到此打开Xcode，会弹出 Load Bundle , 选择 Load Bundle 即可，到此Alcatraz 安装完毕加工作正常，看下图：






Step4:   安装自己想要的插件，以VVDocumenter 为例说一下，因为还要注意一个地方，就是在安装完 VVDocumenter 后，需要在 VVDocumenter 中的 plist 添加 Xcode UUID

如何添加： 找到VVDocumenter 插件 —> 右击显示包内容 —>编辑plist  ,添加Xcode 的 UUID  —> 关闭Xcode（Command+Q）,再打开 Xcode —>  弹出 Load Bundle，选择 Load Bundle ,之后插件正常使用

如果不添加UUID，安装了插件也不能使用。如果再安装其他插件的话也需要重复 Step4 操作 添加UUID 插件才能正常使用
